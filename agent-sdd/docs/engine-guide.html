Overview

The SDD wizard is a web-based tool that auto-detects your mobile project's stack and pre-fills a configuration spec-kit. It reads key files (Gradle, Xcode, source code) and extracts framework signals to recommend the right architecture, testing patterns, and migration rules.

Architecture

Three main components:

  • wizard-core.js (1500+ lines) — Platform-agnostic UI, state, file I/O, sidebar
  • platform-android.js (3200+ lines) — Android-specific detection and form builders
  • platform-ios.js (2900+ lines) — iOS-specific detection and form builders

Key Concepts

  • File I/O: All reads/writes use tryReadFile() which returns file text or null
  • State: Selected folder handle stored in state.dirHandle (the project root)
  • Detection: analyzeProject() reads files, detects frameworks, pre-fills forms
  • Persistence: All form values saved to localStorage (draft persistence)
  • Output: Generated files saved to state.dirHandle/agent-artifacts/ (project folder only)
💡 Important: agent-artifacts/ is created only inside the selected project folder, never inside the agent-sdd folder itself.

File Reading Pattern

const [gradle, settings] = await Promise.all([
  tryReadFile(handle, 'build.gradle'),
  tryReadFile(handle, 'settings.gradle'),
]);
// Returns file text if found, null if not (silent fallback)

Form Pre-filling Pattern

// Direct field assignment
document.getElementById('cfg-min-sdk').value = '24';

// Pill selection (radio/checkbox)
selectPill('cfg-lang', 'Kotlin', 'radio');

// Dropdown selection
setSelectOption('test-runner', 'JUnit 5');

Android Platform Detection

Files Read (in order)

1. Root Gradle Files (Parallel)

  • build.gradle OR build.gradle.kts
  • settings.gradle OR settings.gradle.kts

Used for: Dependency detection, module discovery

2. App Module Gradle

Tries in order:

  1. app/build.gradle (or .kts)
  2. If not found, scans each module in settings.gradle until one has com.android.application

Extracts: applicationId, minSdk, targetSdk, productFlavors

3. Per-Module Gradle Files (Parallel)

One file per Gradle module discovered in settings.gradle

Extracts: Module type (app vs library), DI framework presence

Special: productFlavors Parsing

Simple regex fails because it stops at the first }. Solution: brace-counting

// Navigate through string, counting { and }
let depth = 0;
for (let i = openAt; i < text.length; i++) {
  if (text[i] === '{') depth++;
  else if (text[i] === '}') {
    depth--;
    if (depth === 0) { /* found closing brace */ }
  }
}

For each flavor, extract:

  • Flavor name: development, staging, production
  • applicationIdSuffix → append to base applicationId
  • applicationId → use as-is

Multi-Module Fallback

If app/build.gradle doesn't exist:

const moduleNames = [...settings.matchAll(/include\s*['"]:?([\w\-]+)['"]/g)]
  .map(m => m[1]);

for (const mod of moduleNames) {
  const candidate = await tryReadFile(dirHandle, mod, 'build.gradle');
  if (candidate && /com\.android\.application/.test(candidate)) {
    appGradle = candidate; break;
  }
}

iOS Platform Detection

Files Read (in order)

1. Locate .xcodeproj Directory

Scans project root for *.xcodeproj folder (recursive). Uses found directory for next reads.

2. Dependency Manifests (Parallel)

  • Package.swift — SPM dependencies
  • Podfile — CocoaPods dependencies
  • <App>.xcodeproj/project.pbxproj — Xcode build configuration

3. Swift Source Files (Recursive scan, depth 5)

Reads up to 40 .swift files to detect framework imports and patterns. Skips: .build/, .git/, Pods/, DerivedData/, build/, .swiftpm/

4. Parse project.pbxproj for Bundle IDs

Extracts PRODUCT_BUNDLE_IDENTIFIER from each isa = XCBuildConfiguration; block. Stores in map: configName → bundleId

5. Scan .xcconfig Files (Recursive, depth 6)

Finds BUNDLE_ID or PRODUCT_BUNDLE_IDENTIFIER values in .xcconfig files. Merges with pbxproj map.

6. Resolve Main Bundle ID

Priority order:

  1. AppStore
  2. AppStore-Release
  3. Release
  4. First non-dev ID
  5. First ID if all are dev

Why Multiple Files?

  • pbxproj: Official Xcode config, but may use variable references
  • xcconfig: Fallback for variable definitions
  • Swift source: Framework imports are most reliable signal for actual dependencies

To Add New Framework Detection

  1. Determine the signal source:
    • SPM: hasImp('FrameworkName') — looks for import statement
    • Pods: hasSrc('framework-name') — substring in Podfile
    • Source pattern: hasPat('pattern-key') — regex in Swift files
  2. Open platform-ios.js, find analyzeProject() around line 290
  3. Add detection and form pre-fill: 
    if (hasImp('Kingfisher')) selectPill('img-lib', 'Kingfisher', 'radio');
  4. Update the dependency signals table in this guide

Extending the Wizard

Adding a New Platform (e.g., Flutter, React Native)

  1. Create engine/platform-newos.js
  2. Export PLATFORM object: 
    const PLATFORM = {
  id: 'newos',
  name: 'NewOS',
  steps: [{ id: 'welcome', ... }, ...],
  buildScreens: { projectconfig: ..., ... },
  onFolderGranted: () => analyzeProject(),
  taskDefaults: { qualityGate: [...] },
};
  3. Implement analyzeProject() following Android/iOS pattern:
    • Read key files (build config, manifests, source files)
    • Extract project data via regex/parsing
    • Pre-fill form via selectPill(), document.getElementById().value = ...
    • Call updatePreview() when done
  4. Update setup-wizard.html to add platform choice in overlay

Adding a Form Field

  1. Add HTML input to buildProjectConfigScreen(): 
    <input type="text" id="cfg-fieldname"
  placeholder="e.g. com.example.app"
  oninput="updatePreview('projectconfig'); saveDraft()">
  2. Auto-populate in analyzeProject(): 
    const value = appGradle.match(/your-regex/)?.[1];
const el = document.getElementById('cfg-fieldname');
if (el && !el.value) el.value = value;
  3. Include in MD generation in generateProjectConfigMD(): 
    const fieldname = document.getElementById('cfg-fieldname')?.value.trim();
if (fieldname) md += `fieldname: ${fieldname}\n`;

Common Regex Patterns

Gradle (Android)

// applicationId
const match = gradle.match(/\bapplicationId\b\s*=?\s*["']([^"']+)["']/);

// minSdk / targetSdk
const minMatch = gradle.match(/\bminSdk(?:Version)?\b\s*=?\s*(\d+)/);

// Module names in settings.gradle
const modules = [...gradle.matchAll(/include\s*['"]:?([\w\-]+)['"]/g)]
  .map(m => m[1]);

Xcode (iOS)

// Bundle ID from pbxproj
const match = text.match(/PRODUCT_BUNDLE_IDENTIFIER\s*=\s*([^\s$(\n;]+)\s*;/);

// Build configuration name
const nameMatch = text.match(/\bname\s*=\s*"?([^";\n]+)"?\s*;/);

// Import statement from .swift
const imports = [...text.matchAll(/^import\s+(\w+)/gm)]
  .map(m => m[1]);

Troubleshooting

Problem Likely Cause Fix
Form fields not pre-filled analyzeProject() not called or early return Check that platform's onFolderGranted() is wired
Blank preview panel updatePreview() not called Add updatePreview(stepId) after form updates
Module detection wrong Gradle files not read OR regex mismatch Add console.log after file reads; test regex on actual build.gradle
Old values persist Draft localStorage not cleared Reopen in new browser tab or clear localStorage
iOS bundle ID missing pbxproj/xcconfig not found OR parsing failed Verify app has build configurations; check xcconfig exists

Debugging Steps

  1. Open DevTools (F12), check Console tab for errors
  2. Add logging after each tryReadFile(): 
    const file = await tryReadFile(...);
console.log('Read file:', file ? file.slice(0, 100) : 'NOT FOUND');
  3. Check form state: DevTools → Elements, verify value and checked attributes
  4. Check preview: Does Markdown on right match detected values?
  5. Test edge cases: Multi-module projects, mixed Kotlin+Java, multiple build configs

Code Review Checklist

  • [ ] Regex tested on both Groovy and Kotlin DSL (Android)
  • [ ] Fallbacks in place (try multiple syntax variants)
  • [ ] updatePreview() called after form updates
  • [ ] saveDraft() called for user-editable fields
  • [ ] Dependency signals are case-insensitive
  • [ ] Console has no error/warning messages
  • [ ] Multi-module projects tested
  • [ ] Documentation updated
